# Categories

This list is organized around the infrastructure stack required by production AI agents.

## Agent Runtime

Frameworks and systems for stateful orchestration, tool use, handoffs, workflows, planning, and multi-agent control.

## Execution Sandbox

Infrastructure that lets agents safely run code, shell commands, file operations, notebooks, tests, or development environments.

## Browser Automation

Libraries and hosted platforms for browser sessions, page understanding, Playwright execution, web task automation, and computer-use agents.

## Tool Protocol

Protocols, registries, and schemas that standardize how agents discover and call tools, resources, prompts, and actions.

## App Integrations

OAuth, SaaS connectors, durable jobs, API workflows, event triggers, and integration layers that allow agents to act in external systems.

## Memory And Context

Long-term memory, vector databases, retrieval systems, conversation memory, user profiles, and context stores.

## Safety And Evals

Guardrails, prompt-injection protection, red teaming, validation, test harnesses, and evaluation systems for agent behavior.

## Model Gateway

Unified model access, provider routing, policy control, budget management, fallbacks, logging, and model abstraction.

## Observability

Tracing, prompt logs, span inspection, cost tracking, latency analytics, datasets, evals, and production debugging.

## Deployment And Compute

Serverless compute, GPUs, model serving, workers, queues, containers, regional deployment, and runtime hosting.
