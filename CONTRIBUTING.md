# Contributing

Thanks for helping improve Awesome Agent Runtime.

This repository focuses on infrastructure for production AI agents. Before submitting a project, please check whether it fits the scope.

## Good Fits

- Agent runtimes, orchestration frameworks, and multi-agent systems.
- Sandboxed execution for code, shell, files, notebooks, browser sessions, or dev environments.
- Tool protocols, MCP servers, tool registries, and action schemas.
- Browser automation, computer-use agents, and web task execution.
- Agent memory, context, retrieval, and long-term state.
- Safety, evals, red teaming, prompt-injection defense, and guardrails.
- Observability, tracing, cost analytics, datasets, and debugging.
- Model gateways, routing, policy, budgets, and provider abstraction.
- Deployment, compute, GPU, queues, workers, and serving platforms for agents.

## Not A Good Fit

- Generic AI tool directories.
- Prompt collections with no infrastructure component.
- Thin wrappers around a model API without clear runtime or production value.
- Pure consumer apps unless the underlying agent infrastructure is visible.
- Projects with no public docs, repo, product page, demo, or technical explanation.

## Submission Format

Please open an issue and include:

- Project name.
- Website or GitHub URL.
- Category.
- One-line description.
- Why it matters for production AI agents.
- Useful links such as docs, examples, launch post, or benchmark.

If accepted, the project will be added to `data/projects.json` and the relevant README category.

## Editorial Notes

Inclusion is not a ranking or endorsement. The goal is to map useful infrastructure for builders.

Maintainers may rewrite descriptions for clarity, remove low-signal entries, and reorganize categories as the ecosystem changes.
