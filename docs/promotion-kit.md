# Awesome Agent Runtime Promotion Kit

Use this kit when sharing Awesome Agent Runtime with builders, maintainers, and developer communities.

The project should be promoted as a useful ecosystem map first. SandBase should appear as the maintainer, not the main pitch.

## One-Line Positioning

```text
A curated map of 500 projects across the production AI agent infrastructure stack.
```

## Short Description

```text
Awesome Agent Runtime maps the infrastructure behind production AI agents: runtimes, sandboxes, browser automation, MCP/tool protocols, app integrations, memory, safety/evals, observability, model gateways, and deployment.
```

## Primary Link

```text
https://github.com/sandbaseai/awesome-agent-runtime
```

## Hacker News / Reddit Angle

Prefer resource-first titles:

```text
Show HN: A curated map of 500 AI agent infrastructure projects
```

```text
I mapped 500 projects across the production AI agent infrastructure stack
```

Avoid product-first titles such as:

```text
SandBase launched a new project
```

## Maintainer Outreach

Use this only when a project is already listed or clearly belongs in the map.

```text
Hi, we included {project} in Awesome Agent Runtime, a curated map of production AI agent infrastructure projects:

https://github.com/sandbaseai/awesome-agent-runtime

I placed it under {category} with this short description:
"{description}"

If the category or wording is off, happy to correct it. We want the map to be useful and neutral for builders.
```

## Promotion Rules

- Lead with the resource, not SandBase.
- Invite corrections from maintainers.
- Do not imply listed projects endorse SandBase.
- Avoid rankings unless there is a clear methodology.
- Prefer category insights over generic announcements.
- When posting in communities, add context before linking.
